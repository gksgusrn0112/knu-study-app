## 1. 서비스 요약
### 1.1 서비스명
KNUSTUDY (강남스터디)

### 1.2 한 줄 정의
AI 기반 자동 입력과 학습 시각화를 통해 대학생의 **시간표/일정/할 일/공부 시간/커뮤니티**를 한 곳에서 관리하는 통합 학사·학습 관리 앱

### 1.3 핵심 가치(사용자 가치)
- **입력 자동화**: 시간표 사진 분석, 자연어 일정 생성으로 입력 부담을 줄인다.
- **학습 가시화**: 오늘의 수업/마감/D-Day/목표 달성도를 직관적으로 보여준다.
- **실행 중심**: 할 일 생성 → 공부 측정 → 완료 확인까지 흐름을 연결한다.
- **학교 생활 통합**: 개인 학습 관리 + 익명 커뮤니티를 하나의 앱에서 제공한다.

### 1.4 문제 정의
- 학사/과제/공부/커뮤니티가 여러 앱에 분산되어 **관리 비용이 높고 누락 위험이 큼**
- 일정 앱은 입력이 번거롭고, 공부 앱은 학사 일정과 연결이 약하며, 커뮤니티 앱은 개인 루틴 관리와 분리됨

## 2. 목표/비목표(MVP 범위)
### 2.1 목표
- 시간표/일정/할 일을 쉽고 빠르게 입력할 수 있는 UX 제공
- 홈에서 오늘 해야 할 행동을 즉시 파악 가능
- 공부 측정 및 통계로 자기주도 학습 유도
- 익명 커뮤니티로 정보 교류 활성화
- MVP 기준: **안정적인 CRUD + AI 보조 입력 경험** 제공

### 2.2 성공 지표(KPI 예시)
- 회원가입 후 24시간 이내 시간표 등록률 **60%+**
- 첫 주 내 Todo 1개 이상 생성 비율 **70%+**
- WAU 중 공부 측정 기능 사용률 **40%+**
- 검색 사용 후 게시글 클릭률 **25%+**
- AI 자연어 일정 생성 성공률 **85%+**
- 크래시 프리 세션 **99%+**

### 2.3 비목표(MVP 제외)
- 실시간 채팅, 음성 입력
- 학교 포털/학사 시스템 직접 연동
- 웹 버전
- 반복 일정(추후), 고도화 추천/개인화, 신고/제재 관리자 콘솔 고도화

## 3. 타깃 사용자 및 페르소나
### 3.1 핵심 사용자
대학생(학기 단위로 시간표/과제/시험/팀플을 관리하고, 공부 시간을 기록하며, 익명 커뮤니티를 활용하는 사용자)

### 3.2 페르소나
- **A. 일정 관리형**: 일정 누락이 가장 큰 문제, 캘린더/D-Day 확인 빈도 높음
- **B. 공부 루틴형**: 매일 공부 시간을 측정하고 통계(과목/주간)를 보고 싶음
- **C. 커뮤니티 활용형**: 질문/모집/정보 공유, 검색·카테고리 필터·익명성이 중요

## 4. 핵심 사용자 시나리오(End-to-End)
### 4.1 시간표 등록(사진 기반)
1) 시간표 탭 진입 → 2) AI 사진 업로드 → 3) Gemini Vision이 과목/요일/시간/강의실 추출  
4) 사용자 검수/수정 → 5) 저장 → 6) 주간 시간표 반영

### 4.2 자연어로 일정/할 일 생성
1) 캘린더 탭 → 2) “내일 오후 9시까지 과제 제출” 입력 → 3) Gemini Pro 구조화(JSON)  
4) 프리뷰(제목/날짜/시간/카테고리) → 5) 사용자 확인 후 저장 → 6) schedules/todos 반영

### 4.3 공부 시간 기록
1) 공부 측정 탭 → 2) 할 일 선택 또는 바로 시작 → 3) 시작/일시정지/재개/종료  
4) 종료 시 로그 저장 → 5) 홈/통계에 반영

### 4.4 커뮤니티 탐색/작성
1) 커뮤니티 탭 → 2) 검색/필터 → 3) 게시글 상세 → 4) 댓글 작성(알림 생성)

## 5. 기능 요구사항(모듈별)
### 5.1 인증/프로필
- **목적**: 사용자별 데이터를 안전하게 분리하고 개인화의 기반을 제공
- **요구사항**
  - Supabase Auth 기반 회원가입/로그인/로그아웃(이메일 최소 1개 방식)
  - 최초 가입 시 `profiles` 자동 생성
  - 프로필: 닉네임/학교/학과/학년/목표 공부 시간(분) 설정
  - 인증 실패 시 사용자 친화적 오류 메시지
- **수용 기준**
  - 로그인 사용자만 개인 데이터 접근 가능

### 5.2 홈(학습 대시보드)
- **구성**: 오늘의 수업, 마감/D-Day, 오늘의 Todo 진행률, 공부 시간 게이지, 최근 요약, 빠른 액션
- **요구사항**
  - 오늘 날짜 기준 필터링이 정확해야 함
  - 데이터 없음/로딩/에러 상태 UI 제공
- **수용 기준**
  - 홈 진입 시 핵심 요약이 빠르게 표시(목표: 2초 이내)

### 5.3 시간표(주간 반복 수업)
- **CRUD**
  - Create: 직접 입력 / AI 이미지 분석 등록(검수 필수)
  - Read: 주간 그리드
  - Update: 강의실/시간/과목명/메모/색상
  - Delete: 종강/취소 과목 제거
- **검증/예외**
  - 종료시간 > 시작시간 필수
  - 동일 시간대 중복 시 경고
  - 인식 실패 시 수동 입력으로 즉시 전환
- **수용 기준**: 주간 뷰 배치 정확, 수정/삭제 즉시 반영, 잘못된 시간 저장 불가

### 5.4 Todo(실행 중심 작업)
- **요구사항**
  - 제목 필수, 설명/마감/카테고리/우선순위는 선택
  - 완료 토글 및 `completed_at` 저장
  - 마감일 없는 Todo 허용
  - 일정(`schedules`)과 연결 가능
- **수용 기준**: CRUD 및 완료 변경이 즉시 UI/통계에 반영

### 5.5 캘린더/일정(날짜 기반 이벤트)
- **요구사항**
  - 월간 캘린더 + 선택 날짜의 일정/관련 Todo 리스트
  - 일정: 시작/종료/종일 여부, 카테고리, Todo 생성/연결
  - 반복 일정은 MVP 제외(확장 여지 확보)
- **수용 기준**: 날짜 필터 정확, 월간/리스트 일관성

### 5.6 AI 할 일/일정 생성(Gemini)
- **목적**: 자연어 입력을 구조화하여 입력 편의성 향상
- **처리 흐름**
  - 자연어 입력 → Gemini 호출 → JSON 응답 → 스키마 검증(Zod 등) → 프리뷰 → 사용자 확인 후 저장
- **요구사항**
  - 제목/날짜/시간/카테고리 추출
  - 상대 날짜(오늘/내일/모레/다음 주 월요일) 처리
  - 날짜/시간 불명확 시 저장 전 사용자 확인 요구
  - 응답 실패/포맷 오류 시 수동 입력 fallback
  - 민감/무관 텍스트는 저장하지 않도록 후처리
- **수용 기준**: 스키마 불일치 응답은 저장되지 않으며, 자동 저장 금지(검수 필수)

### 5.7 공부 측정/통계
- **요구사항**
  - 스톱워치: 시작/일시정지/재개/종료(동시 1개 세션)
  - 백그라운드 복귀/앱 종료에 대비한 세션 복원(로컬 임시 상태)
  - `study_logs` 저장(중복 저장 방지)
  - 통계: 일/주 누적, 과목/카테고리별, 평균 세션 길이, 목표 대비 달성률
- **수용 기준**: 로그 합계와 통계가 일치, 복원 동작 일관

### 5.8 익명 커뮤니티
- **기능**: 게시글/댓글 CRUD, 검색(제목+본문), 최신순, 카테고리 필터, 상세, 알림 연동
- **정책**
  - 작성자는 익명 표시(내부적으로 `user_id`로 추적)
  - 본인 글/댓글만 수정·삭제 가능
  - 삭제는 소프트 삭제 권장
- **수용 기준**: 작성 후 리스트 상단 반영, 권한 없는 수정/삭제 불가, 검색/필터 동작

### 5.9 알림
- **MVP**: 댓글 알림, 좋아요 알림(도입 시)
- **요구사항**: 읽음/안읽음, 클릭 시 관련 화면 이동(딥링크), 중복 최소화
- **수용 기준**: 이벤트 발생 시 알림 레코드 생성, 읽음 처리 UI 반영

## 8. 데이터 모델/관계(Supabase)
### 8.1 테이블
- `profiles`: 사용자 기본 정보(목표 공부 시간 포함)
- `todos`: 실행 중심 작업(완료/우선순위/카테고리/AI source)
- `schedules`: 날짜 기반 일정(종일/기간)
- `timetables`: 주간 반복 수업(요일/시간/학기 기간)
- `study_logs`: 공부 세션 로그(초 단위 duration)
- `posts`, `comments`: 커뮤니티
- `notifications`: 댓글/좋아요 등 알림

### 8.2 관계(요약)
- `profiles.id = auth.users.id`
- 소유 관계: `todos/schedules/timetables/study_logs/posts/comments/notifications.user_id → profiles.id`
- 연결: `study_logs.todo_id → todos.id`, `comments.post_id → posts.id`

## 9. 권한/보안 정책
- **RLS 필수**: 개인 데이터는 본인만 접근 가능
- **커뮤니티**: 읽기는 정책에 따라 공개/회원 제한 선택 가능(수정/삭제는 작성자만)
- **Storage**: 사용자 권한 기준 접근 제어
- **AI 호출**: 클라이언트 직호출보다 Edge Functions/서버 중계를 권장(키/요금/보안)
- **익명성**: 외부 표시 익명, 내부 추적은 `user_id` 유지(운영 대응 목적)

## 10. 비기능 요구사항
- **성능**: 주요 탭 첫 진입 로딩 최소화, 리스트 스크롤 안정
- **안정성**: 공부 세션 유실 최소화(복원/안내), 네트워크 불안정 시 재시도/오프라인 안내
- **확장성**: 반복 일정/인기 정렬/학교별 커뮤니티 분리 등을 고려한 설계
- **접근성**: 명확한 레이블, 색상 외 텍스트/아이콘으로 상태 표현, 작은 화면 최적화

## 11. 예외 처리/에러 정책(대표 케이스)
- 시간표 AI 인식 실패: 실패 안내 → 수동 입력으로 즉시 전환
- 자연어 인식 실패: 날짜/시간 확인 요청 → 제목 축약 제안 → 수동 입력 전환
- 공부 중 앱 종료: 마지막 시작 시각 로컬 저장 → 재실행 시 복원/종료 안내
- 권한 오류: 수정/삭제 차단, 세션 만료 시 재로그인 유도

## 13. 출시/개발 우선순위
### 13.1 MVP 필수
로그인/회원가입, 홈 요약, 시간표 CRUD, Todo CRUD, 캘린더 CRUD, AI 자연어 생성, 공부 측정+기본 통계, 커뮤니티 CRUD, 검색/필터/최신순, 알림 기본

### 13.2 2차 확장
푸시 알림, 반복 일정, 인기순, 신고 기능, 목표 추천, 학교/학과 기반 커뮤니티 세분화

### 13.3 개발 단계(제안)
- 1단계: 인증/프로필/스키마/내비게이션
- 2단계: 시간표 + Todo/캘린더 + 홈 연동
- 3단계: 공부 측정 + `study_logs` + 통계
- 4단계: 커뮤니티 + 검색/필터/정렬 + 알림
- 5단계: Gemini 연동(비전/자연어)

## 14. 완료 정의(Definition of Done, MVP)
- 회원가입/로그인/프로필 생성 가능
- 시간표 직접 입력 및 AI 등록 가능(검수 포함)
- Todo/일정 CRUD 및 완료 처리 가능
- 자연어 입력을 일정/Todo로 변환 가능(스키마 검증+프리뷰+검수)
- 공부 시간 측정/로그 확인/기본 통계 확인 가능
- 홈에서 오늘의 수업/일정/D-Day/Todo/공부 현황 확인 가능
- 커뮤니티 게시글/댓글 CRUD 및 검색/필터/정렬 동작
- 기본 알림 동작
- 주요 데이터에 RLS/권한 정책 적용
